# Short Bot #2 — Шорт #2 монеты после отката с #1

> Paper trading бот для Bybit USDT-perpetual. Стратегия основана на анализе
> 923 входов за 180 дней тиковых данных (2026-01-04 → 2026-07-02).

## 📌 Идея (Сценарий Б)

**Сигнал:** монета **впервые** стала #1 по росту среди всех USDT perp
(от open дня) → откатилась на #2 → **шортим #2**.

**Логика:** монета, только что вставшая #1 по росту — это свежий памп.
Когда она теряет лидерство и падает на #2,大概率 начинается откат/крах.
Шорт #2 ловит движение вниз.

**Ключевое отличие от лонга:** лонг #2 контр-трендовый (памп-дамп),
а шорт #2 идёт по естественному движению — монета уже теряет момент.

## 🔧 Как работает бот

### Архитектура

```
Bybit WS (kline.1 + tickers, 416 символов)
    ↓
Ранжирование каждые 15 мин (рост от day open)
    ↓
Сигнал: #1 впервые → откат на #2 → шорт #2
    ↓ (бан монеты на 24ч)
Paper executor: комиссия 0.055% + проскальзывание 0.02% + фандинг 0.01%/8ч
    ↓
Position sizing: risk_pct / sl_pct → qty
    ↓
SL/TP/Trail мониторинг по 1m свечам
    ↓
Логи: trades.jsonl, positions.json, equity.jsonl
```

### Параметры (config.yaml)

```yaml
direction: short
mode: paper
risk_pct: 2.0           # % депозита на сделку (настраиваемо)
initial_balance: 1000.0
sl_pct: 30.0            # стоп (монета растёт против шорта)
tp_pct: 15.0            # тейк (монета падает)
trail_pct: 10.0         # трейлинг отступ
activation_pct: 1.0     # активация trail
commission_taker_pct: 0.055
slippage_pct: 0.02
funding_rate_avg: 0.0001
ranking_interval_min: 15
cooldown_symbol_hours: 24
```

### Деплой

```bash
# VPS 82.148.19.150
/opt/short_bot/
  ├── bot.py           # основной модуль
  ├── config.yaml      # конфиг
  ├── short-bot.service # systemd
  └── logs/
      ├── bot.log
      ├── trades.jsonl
      ├── positions.json
      └── equity.jsonl

systemctl status short-bot   # статус
systemctl restart short-bot  # после config изменения
```

## ⚠️ Важно

- **Paper trading** — деньги виртуальные ($1000 фантиков), реальные ордера не отправляются
- **Цены/фандинги реальные** с Bybit — симуляция честная
- **Compounding цифры теоретические** — на реале ликвидность и проскальзывание выше
- Перед live: проверить paper результаты 2-3 дня, потом переключить `mode: live`
